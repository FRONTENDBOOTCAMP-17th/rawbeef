## 3-1. Tailwind CSS CDN → Vite 플러그인으로 통합 (중요도: 높음)

### 문제

`soyu/` 폴더의 HTML 파일 4개가 모두 CDN 방식(`<script src="https://cdn.tailwindcss.com">`)을 사용하고 있다. 프로젝트에 이미 Vite + `@tailwindcss/vite` 플러그인이 설치되어 있는데 활용하지 않는 상태.

### 해결 방법

**Step 1.** `vite.config.js`에 `soyu` 폴더를 빌드 대상에 추가한다.

```js
// vite.config.js (38~40번째 줄)
// 수정 전
input: {
  index: path.resolve(__dirname, "index.html"),
  ...findAllHtmlFiles(path.resolve(__dirname, "src")),
},

// 수정 후
input: {
  index: path.resolve(__dirname, "index.html"),
  ...findAllHtmlFiles(path.resolve(__dirname, "src")),
  ...findAllHtmlFiles(path.resolve(__dirname, "soyu")),  // 추가
},
```

**Step 2.** `soyu/` 폴더의 HTML 파일 4개에서 CDN 스크립트를 삭제하고, Tailwind CSS 파일을 import한다.

각 HTML 파일의 `<head>` 안에서:

```html
<!-- 삭제할 줄 -->
<script src="https://cdn.tailwindcss.com"></script>

<!-- 대신 추가할 줄 -->
<link rel="stylesheet" href="../src/style.css" />
```

수정 대상 파일 4개:
- `soyu/admin_open.html` (7번째 줄)
- `soyu/admin_category.html` (7번째 줄)
- `soyu/admin_song.html` (7번째 줄)
- `soyu/admin_request.html` (7번째 줄)

> **참고:** `src/style.css`에 이미 `@import "tailwindcss"` 등 Tailwind 설정이 되어 있으므로, 이 파일을 연결하면 Vite가 빌드 시 자동으로 Tailwind를 처리해준다.

## 3-2. 사이드바 코드 반복 제거 (중요도: 높음)

### 문제

4개의 HTML 파일 모두에 동일한 사이드바 `<aside>` 코드가 복사-붙여넣기 되어 있다. 메뉴를 수정하려면 4개 파일을 모두 수정해야 한다.

### 해결 방법

사이드바를 JavaScript로 분리하여 하나의 파일에서 관리한다.

**Step 1.** `soyu/sidebar.js` 파일을 새로 만든다.

```js
// soyu/sidebar.js

const menuItems = [
  { href: './admin_open.html', label: '관리자메뉴' },
  { href: './admin_category.html', label: '카테고리관리' },
  { href: './admin_song.html', label: '노래 관리' },
  { href: './admin_request.html', label: '노래 신청 관리' },
];

function createSidebar() {
  // 현재 페이지 파일명 구하기
  const currentPage = './' + location.pathname.split('/').pop();

  const aside = document.createElement('aside');
  aside.className = 'w-1/5 bg-white border-r-4 border-black flex flex-col p-6 gap-6';

  menuItems.forEach((item) => {
    const link = document.createElement('a');
    link.href = item.href;
    link.className = 'block w-full';

    // 현재 페이지면 파란색 강조, 아니면 hover 효과
    const isActive = currentPage === item.href;
    link.className = `block w-full py-4 border-2 border-black font-bold text-center ${
      isActive ? 'bg-blue-600 text-white' : 'hover:bg-gray-200'
    }`;

    link.textContent = item.label;
    aside.appendChild(link);
  });

  return aside;
}

// 페이지에 사이드바 삽입
document.addEventListener('DOMContentLoaded', () => {
  const container = document.getElementById('sidebar-container');
  if (container) {
    container.replaceWith(createSidebar());
  }
});
```

**Step 2.** 각 HTML 파일에서 기존 `<aside>...</aside>` 전체를 삭제하고, 대신 placeholder와 스크립트를 넣는다.

```html
<!-- 기존 <aside>...</aside> 전체를 이것으로 대체 -->
<div id="sidebar-container"></div>

<!-- body 닫기 전에 추가 -->
<script src="./sidebar.js"></script>
```

> **효과:** 이제 메뉴를 추가하거나 이름을 바꿀 때 `sidebar.js` 한 파일만 수정하면 된다. 또한 `<a>` 안에 `<button>`을 넣는 문제(3-3)도 자연스럽게 해결된다.

## 3-3. `<a>` 태그 안에 `<button>` 넣기 제거 (중요도: 높음)

### 문제

```html
<!-- 현재 코드 (HTML 규칙 위반) -->
<a href="./admin_open.html" class="block w-full">
  <button class="w-full py-4 border-2 border-black font-bold hover:bg-gray-200">관리자메뉴</button>
</a>
```

`<a>` 안에 `<button>`을 넣는 것은 HTML 표준에 위배된다. 둘 다 인터랙티브 요소이기 때문.

### 해결 방법

`<a>` 태그에 버튼 스타일을 직접 적용하고, `<button>`을 제거한다.

```html
<!-- 수정 후 (admin_open.html, admin_category.html, admin_song.html, admin_request.html) -->

<!-- 일반 메뉴 항목 -->
<a href="./admin_open.html"
   class="block w-full py-4 border-2 border-black font-bold text-center hover:bg-gray-200">
  관리자메뉴
</a>

<!-- 현재 활성화된 메뉴 항목 -->
<a href="./admin_category.html"
   class="block w-full py-4 border-2 border-black font-bold text-center bg-blue-600 text-white">
  카테고리관리
</a>
```

수정 대상: 4개 HTML 파일의 사이드바 영역 전부 (파일당 4개 메뉴 항목)

> **참고:** 위의 3-2(사이드바 분리)를 적용하면 이 문제도 함께 해결되므로, 3-2를 먼저 하는 것을 추천한다.

## 3-4. 모달의 `hidden`과 `flex` 충돌 해결 (중요도: 중간)

### 문제

```html
<!-- admin_category.html 39번째 줄, admin_song.html 52번째 줄 -->
<div id="modal" class="fixed inset-0 bg-black hidden flex items-center justify-center z-50">
```

`hidden`(`display: none`)과 `flex`(`display: flex`)가 동시에 선언되어 있어 CSS 우선순위에 따라 예측 불가능하게 동작할 수 있다.

### 해결 방법

**Step 1.** HTML에서 `flex`를 제거한다.

```html
<!-- 수정 전 -->
<div id="modal" class="fixed inset-0 bg-black hidden flex items-center justify-center z-50">

<!-- 수정 후 (flex 제거) -->
<div id="modal" class="fixed inset-0 bg-black hidden items-center justify-center z-50">
```

**Step 2.** JavaScript에서 모달을 열고 닫을 때 `flex`를 함께 토글한다.

`admin_category.html`의 JavaScript를 수정한다:

```js
// 모달 열기 (카테고리 추가 버튼 - 69~72번째 줄)
addCategoryBtn.addEventListener('click', () => {
  editIndex = null;
  modal.classList.remove('hidden');
  modal.classList.add('flex');       // 추가
});

// 모달 열기 (EDIT 버튼 - 111~116번째 줄의 editBtn 이벤트 안)
editBtn.addEventListener('click', function () {
  editIndex = index;
  document.querySelector('#modal h2').textContent = '카테고리 수정';
  document.getElementById('titleInput').value = item.title;
  modal.classList.remove('hidden');
  modal.classList.add('flex');       // 추가
});

// 모달 닫기 (닫기 버튼 - 75~77번째 줄)
closeCategoryBtn.addEventListener('click', () => {
  modal.classList.add('hidden');
  modal.classList.remove('flex');    // 추가
});

// 모달 닫기 (저장 후 - 171번째 줄)
modal.classList.add('hidden');
modal.classList.remove('flex');      // 추가
```

`admin_song.html`도 동일하게 수정한다:

## 3-6. `admin_category.html` - `saveCategory()` 호출 누락 수정 (중요도: 중간)

### 문제

카테고리를 추가하거나 수정한 뒤 `saveCategory()`를 호출하지 않아서, 페이지를 새로고침하면 변경 내용이 사라진다. 또한 삭제 시에도 `saveCategory()`가 빠져 있다.

### 해결 방법

**수정 1.** 저장 버튼 이벤트에 `saveCategory()` 추가 (170번째 줄 근처):

```js
// 수정 전 (159~177번째 줄)
saveCategoryBtn.addEventListener('click', () => {
  const titleInput = document.getElementById('titleInput');
  const title = titleInput.value;

  if (title) {
    if (editIndex !== null) {
      category[editIndex].title = title;
      editIndex = null;
    } else {
      category.push({ title: title });
    }
    renderCategory();
    // <-- 여기에 빠져 있음
    modal.classList.add('hidden');
    titleInput.value = '';
    document.querySelector('#modal h2').textContent = '신규 카테고리 등록';
  } else {
    alert('내용이 입력되지 않았습니다!');
  }
});

// 수정 후 — renderCategory() 다음 줄에 saveCategory() 추가
saveCategoryBtn.addEventListener('click', () => {
  const titleInput = document.getElementById('titleInput');
  const title = titleInput.value;

  if (title) {
    if (editIndex !== null) {
      category[editIndex].title = title;
      editIndex = null;
    } else {
      category.push({ title: title });
    }
    renderCategory();
    saveCategory();  // 추가!
    modal.classList.add('hidden');
    titleInput.value = '';
    document.querySelector('#modal h2').textContent = '신규 카테고리 등록';
  } else {
    alert('내용이 입력되지 않았습니다!');
  }
});
```

## 3-7. `admin_song.html` - score가 0일 때 저장 안 되는 버그 수정 (중요도: 중간)

### 문제

```js
// admin_song.html 168번째 줄
if (title && artist && score) {
```

JavaScript에서 숫자 `0`은 falsy 값이므로, `score`에 0을 입력하면 `if` 조건이 `false`가 되어 "내용이 입력되지 않았습니다!" 알림이 뜬다.

### 해결 방법

`score`의 유효성을 별도로 체크한다.

```js
// 수정 전 (168번째 줄)
if (title && artist && score) {

// 수정 후
if (title && artist && score !== null && !isNaN(score)) {
```

더 간결하게 쓸 수도 있다:

```js
// 또는 이렇게
if (title && artist && score !== '') {
```

> **설명:** `score`는 이미 `parseInt()`로 변환되므로, `!isNaN(score)`로 체크하면 숫자 `0`도 유효한 값으로 인정된다. 빈 문자열을 `parseInt()`하면 `NaN`이 되므로 빈 입력도 걸러진다.

## 3-10. `<label>`과 `<input>` 연결하기 (중요도: 중간)

### 문제

`<label>`에 `for` 속성이 없어서 `<input>`과 연결되지 않았다. label을 클릭해도 input에 포커스가 가지 않고, 스크린 리더에서도 어떤 입력 칸인지 인식하기 어렵다.

### 해결 방법

**`admin_category.html`** (45~46번째 줄):

```html
<!-- 수정 전 -->
<label class="block font-black mb-1">노래 카테고리</label>
<input type="text" id="titleInput" class="..." />

<!-- 수정 후 — for 속성 추가 -->
<label for="titleInput" class="block font-black mb-1">노래 카테고리</label>
<input type="text" id="titleInput" class="..." />
```

**`admin_song.html`** (58~59, 62~63, 66~67번째 줄):

```html
<!-- 수정 전 -->
<label class="block font-black mb-1">제목</label>
<input type="text" id="titleInput" ... />

<label class="block font-black mb-1">가수</label>
<input type="text" id="artistInput" ... />

<label class="block font-black mb-1">SCORE</label>
<input type="number" id="scoreInput" ... />

<!-- 수정 후 — 각 label에 for 속성 추가 -->
<label for="titleInput" class="block font-black mb-1">제목</label>
<input type="text" id="titleInput" ... />

<label for="artistInput" class="block font-black mb-1">가수</label>
<input type="text" id="artistInput" ... />

<label for="scoreInput" class="block font-black mb-1">SCORE</label>
<input type="number" id="scoreInput" ... />
```
